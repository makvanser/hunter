# Orchestration & Planning Rules

## Plan Node Default

- Для любой задачи, требующей более 3 шагов или изменения архитектуры, сначала войди в режим планирования.
- Опиши план в `tasks/todo.md` перед внесением изменений в код.
- Если в процессе реализации возникают непредвиденные сложности, ОСТАНОВИСЬ и пересмотри план. Не пытайся "пробить" решение силой.

## Subagent Strategy

- Используй субагентов для параллельных задач: исследование документации API (RhinoCommon, Crypto-библиотеки), поиск примеров или аудит безопасности.
- Ограничивай контекст субагента одной конкретной задачей, чтобы избежать галлюцинаций.

## Elegance & Complexity

- Для нетривиальных изменений делай паузу. Спрашивай: "Существует ли более элегантное решение?".
- Приоритет отдается чистоте и поддерживаемости кода, а не скорости написания.
- Для простых исправлений (опечатки, явные баги) пропускай этот этап, чтобы избежать овер-инжиниринга.
